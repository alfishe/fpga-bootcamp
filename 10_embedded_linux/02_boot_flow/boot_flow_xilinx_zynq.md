# Xilinx Zynq Boot Flow — Deep Dive

This article covers the complete boot flow for Xilinx Zynq families: **Zynq-7000**, **Zynq UltraScale+ MPSoC**, and **Versal ACAP**. It details the Boot ROM / CSU ROM behavior, FSBL vs U-Boot SPL, PCAP FPGA configuration, `bootgen` image generation, secure boot with RSA/AES/PUF, and per-board specifics for popular development kits.

---

| Family | CPU | FPGA Fabric | Boot ROM | Key Document |
|---|---|---|---|---|
| **Zynq-7000** | ARM Cortex-A9 dual-core @ 1 GHz | Artix-7 / Kintex-7 FPGA (28 nm) | 128 KB OCM + Boot ROM | UG585, Ch. 6 |
| **Zynq UltraScale+ MPSoC** | Cortex-A53 quad @ 1.5 GHz + Cortex-R5F dual @ 600 MHz | Kintex UltraScale+ / Zynq UltraScale+ (16 nm) | 256 KB CSU ROM | UG1085, Ch. 11 |
| **Versal ACAP** | Cortex-A72 dual @ 1.7 GHz + Cortex-R5F dual | AI Engines + programmable logic (7 nm) | 512 KB PMC ROM | AM011 |

## Boot ROM / CSU ROM — The First Stage

Xilinx uses different first-stage boot mechanisms across generations. The terminology changes, but the function is the same: load the first-stage bootloader from the selected boot medium.

### Zynq-7000 Boot ROM

The Zynq-7000 Boot ROM is a 128 KB on-chip memory (OCM) block that contains the boot code. It executes after power-on reset (POR):

1. **Boot mode sampling**: The Boot ROM reads the `BOOT_MODE` register (set by MIO pin straps at POR).
2. **PLL initialization**: The Boot ROM configures the ARM PLL to a safe frequency (~333 MHz for the CPU).
3. **Load FSBL**: The Boot ROM reads the FSBL (First-Stage Bootloader) from the boot medium into OCM.
4. **Authentication (if secure boot)**: The Boot ROM verifies the FSBL using the RSA public key hash stored in eFuse.
5. **Jump to FSBL**: Control transfers to the FSBL at OCM address `0x0000_0000`.

> **Zynq-7000 OCM is 256 KB total**: Boot ROM uses the first 128 KB. FSBL runs in the remaining 128 KB (or DDR after initialization).

### Zynq MPSoC — CSU ROM + PMU

The Zynq UltraScale+ MPSoC introduces the **Platform Management Unit (PMU)** and the **Configuration and Security Unit (CSU)**:

1. **PMU boots first**: The PMU executes from its own 384 KB ROM. It initializes the power islands and releases the CSU from reset.
2. **CSU ROM executes**: The CSU ROM (256 KB) reads boot mode and loads two images:
   - **PMU firmware** (PMUFW): Executes on the PMU MicroBlaze. Manages power, clocks, and resets.
   - **FSBL/U-Boot SPL**: Executes on the Cortex-A53 (APU) or Cortex-R5F (RPU).
3. **FSBL/U-Boot SPL runs**: Initializes DDR, configures the FPGA (optional), and loads U-Boot proper.

> **MPSoC has three processors at boot time**: PMU MicroBlaze (always first), then CSU, then APU/RPU. This is more complex than Zynq-7000 but enables fine-grained power management.

### Versal — PMC ROM + PLM

Versal introduces the **Platform Management Controller (PMC)** and the **Platform Loader and Manager (PLM)**:

1. **PMC ROM executes**: The PMC ROM (512 KB) is the first code to run. It initializes the NoC (Network on Chip), DDR memory controllers, and PMC peripherals.
2. **PLM loaded**: The PMC ROM loads the PLM (a MicroBlaze application) into PMC RAM.
3. **PLM executes**: The PLM manages the boot of all other processing units:
   - **PSM (Platform State Manager)**: Power and clock management
   - **APU (Cortex-A72)**: Application processing
   - **RPU (Cortex-R5F)**: Real-time processing
   - **PL (Programmable Logic)**: FPGA fabric configuration
   - **AI Engines**: AI Engine array configuration
4. **U-Boot / Linux loaded**: The PLM loads U-Boot or the Linux kernel directly, bypassing a separate FSBL stage.

> **Versal boot is fundamentally different**: The PLM replaces both the FSBL and U-Boot SPL. It is a vendor-provided binary that you configure via the Vivado/Vitis boot image generator (BIF).

### Boot Mode Register Reading

```bash
# Zynq-7000 — read SLCR BOOT_MODE register (0xF800_025C)
devmem 0xF800025C
# Bits [4:0] = BOOT_MODE

# Zynq MPSoC — read CSU_BOOT_MODE register (0xFF5E_0200)
devmem 0xFF5E0200

# Versal — read via XSDB/XSCT
# xsdb% mrd 0xF11A0000  # PMC_BOOT_MODE
```

## FSBL vs U-Boot SPL

Xilinx traditionally uses the **FSBL** (First-Stage Bootloader), a proprietary binary generated by the SDK/Vitis. However, modern designs increasingly use **U-Boot SPL** for better open-source control.

| Feature | Xilinx FSBL | U-Boot SPL |
|---|---|---|
| **Source** | Proprietary (Xilinx SDK/Vitis) | Open-source (U-Boot tree) |
| **Customization** | Limited — C hooks for user code | Full source control |
| **DDR init** | Uses generated `ps7_init.c` / `psu_init.c` | Same init code, integrated in U-Boot |
| **FPGA config** | Built-in PCAP support | Supported via `CONFIG_SPL_FPGA` |
| **Authentication** | Full RSA/ AES support | Supported via Xilinx secure boot hooks |
| **Distro boot** | No | Yes — modern standard |
| **Maintenance** | Tied to Xilinx tool versions | Active open-source community |

> **Recommendation**: Use U-Boot SPL for new designs. It integrates with distro boot, is actively maintained, and avoids vendor tool version lock-in.

### FSBL Responsibilities

```c
// Pseudo-code of FSBL main flow (Xilinx SDK generated)

int main(void) {
    // 1. Initialize CPU (caches, MMU, vectors)
    init_cpu();
    
    // 2. Initialize PLLs and clocks
    ps7_init();  // or psu_init() for MPSoC
    
    // 3. Initialize DDR (MIG/DDRC calibration)
    ps7_ddr_init();
    
    // 4. Initialize MIO pin muxing
    ps7_mio_init();
    
    // 5. Configure FPGA via PCAP (optional)
    #ifdef FPGA_CONFIG
    XFpga_PcapLoadBitstream(bitstream, size);
    #endif
    
    // 6. Load U-Boot or application into DDR
    load_image("u-boot.bin", DDR_BASE_ADDR);
    
    // 7. Hand off
    handoff_to_ddr(DDR_BASE_ADDR);
}
```

### PMU Firmware (MPSoC Only)

The PMU firmware is a separate binary that runs on the PMU MicroBlaze:

```bash
# PMU firmware is loaded by CSU ROM before FSBL
# It must be included in the boot image (BOOT.BIN)

# Typical MPSoC boot image contains:
# 1. PMUFW (pmufw.elf)
# 2. FSBL (fsbl.elf)
# 3. U-Boot (u-boot.elf) or ATF + U-Boot
# 4. Optional: FPGA bitstream
```

## U-Boot for Zynq

U-Boot for Zynq is built with the `zynq-common` or `zynqmp-common` board support.

### Building U-Boot for Zynq-7000

```bash
# Clone U-Boot
git clone https://github.com/u-boot/u-boot.git
cd u-boot

# Configure for ZedBoard (Zynq-7000)
make ARCH=arm zynq_zed_defconfig

# Build
make ARCH=arm CROSS_COMPILE=arm-linux-gnueabihf- -j$(nproc)

# Outputs:
#   spl/boot.bin   →  FSBL + SPL (combined)
#   u-boot.img     →  U-Boot proper
```

### Building U-Boot for Zynq MPSoC

```bash
# Configure for ZCU102 (Zynq UltraScale+ MPSoC)
make ARCH=arm zynqmp_zcu102_rev1_0_defconfig

# Build
make ARCH=arm CROSS_COMPILE=aarch64-linux-gnu- -j$(nproc)

# Outputs:
#   spl/boot.bin   →  FSBL + PMUFW + ATF + SPL
#   u-boot.itb     →  U-Boot proper (FIT image)
```

### U-Boot FPGA Commands (Xilinx PCAP)

```bash
# Load bitstream from FAT partition on SD
fatload mmc 0:1 0x1000000 system.bit.bin
fpga loadb 0 0x1000000 ${filesize}

# For Zynq MPSoC — load via CSU DMA
fatload mmc 0:1 0x1000000 system.bit.bin
fpga load 0 0x1000000 ${filesize}

# Check FPGA status
fpga info 0
# Output: FPGA[0] state: Powered Up / Configured

# Load bitstream from QSPI
sf probe
sf read 0x1000000 0x200000 0x500000
fpga load 0 0x1000000 ${filesize}
```

> **Note**: Xilinx bitstreams for U-Boot FPGA commands must be `.bin` format (raw, no header), not `.bit` format (which has a header). Convert with `bootgen` or use the `-bin_file` option in Vivado write_bitstream.

### Boot Script Generation (`mkimage`)

```bash
# Create boot.cmd
# For Zynq-7000 / ZedBoard

cat > boot.cmd << 'EOF'
# Load bitstream
fatload mmc 0:1 0x1000000 system.bit.bin
fpga loadb 0 0x1000000 ${filesize}

# Load device tree
fatload mmc 0:1 ${fdtaddr} zynq-zed.dtb

# Load kernel
fatload mmc 0:1 ${kernel_addr_r} zImage

# Set bootargs
setenv bootargs console=ttyPS0,115200 root=/dev/mmcblk0p2 rw rootwait

# Boot Linux
bootz ${kernel_addr_r} - ${fdtaddr}
EOF

# Convert to boot.scr
mkimage -A arm -T script -C none -n "ZedBoard Boot" -d boot.cmd boot.scr
```

### Distro Boot Support

```bash
# Modern U-Boot for Zynq supports distro boot
# Scans for extlinux.conf on SD, eMMC, USB, network

# Example /boot/extlinux/extlinux.conf for Zynq MPSoC
LABEL primary
    MENU LABEL Primary Kernel
    LINUX ../Image
    FDT ../xilinx/zynqmp-zcu102-rev1.0.dtb
    INITRD ../initrd.img
    APPEND console=ttyPS0,115200 root=/dev/mmcblk0p2 rw rootwait
```

## FPGA Configuration — PCAP Interface

Xilinx Zynq devices use the **Processor Configuration Access Port (PCAP)** for FPGA configuration. The PCAP is a dedicated DMA engine in the PS (Processing System) that streams the bitstream to the PL (Programmable Logic) configuration logic.

### PCAP Architecture

```
┌─────────────────────────────────────┐
│  Processing System (PS)             │
│  ┌─────────┐   ┌─────────────────┐  │
│  │  CPU    │──→│  PCAP DMA       │  │
│  │  A53/A9 │   │  (CSU_DMA for    │  │
│  │         │   │   MPSoC)         │  │
│  └─────────┘   └────────┬────────┘  │
│                         │           │
│                         ▼           │
│                  ┌─────────────┐    │
│                  │  PL Config  │    │
│                  │  Logic      │    │
│                  └─────────────┘    │
└─────────────────────────────────────┘
```

### PCAP Programming Steps

1. **Enable PCAP**: Write to `devcfg.CTRL` register to enable PCAP.
2. **Clear FIFO**: Reset the PCAP DMA FIFO.
3. **Write bitstream**: The CPU writes bitstream words to `devcfg.DMA_SRC_ADDR` and `devcfg.DMA_DST_ADDR`. The PCAP DMA transfers data to the PL configuration logic.
4. **Poll status**: Wait for `devcfg.STATUS` to indicate configuration complete.
5. **Verify**: Check `devcfg.STATUS[PCAP_CFG_DONE]` bit.

> **Zynq-7000**: PCAP is accessed via the `devcfg` registers (0xF800_7000). **Zynq MPSoC**: PCAP is accessed via the CSU DMA (0xFFCA_0000).

### Bitstream Formats

| Format | Extension | Header | Use With |
|---|---|---|---|
| **BIT** | `.bit` | Yes ( Xilinx header with filename, date, etc.) | Vivado iMPACT, JTAG direct programming |
| **BIN** | `.bin` | No (raw bitstream) | U-Boot `fpga load`, Linux FPGA Manager, embedded flash |
| **MCS** | `.mcs` | Intel HEX-like | Flash programmers (Xilinx Platform Flash) |
| **RBT** | `.rbt` | ASCII text | Rare — debugging only |

```bash
# Generate .bin from Vivado (for U-Boot / embedded use)
write_bitstream -force -bin_file system.bit
# Output: system.bit.bin

# Or use bootgen to create a boot image with bitstream
bootgen -image boot.bif -arch zynq -o BOOT.BIN -w on
```

## `bootgen` — Boot Image Generation

`bootgen` is the Xilinx tool that creates the boot image (`BOOT.BIN`) containing all the binaries needed for boot.

### Boot Image Structure

```
BOOT.BIN
├─ Boot Header (authentication, encryption flags)
├─ Image Header Table (partition table)
├─ Partition 1: FSBL (first-stage bootloader)
├─ Partition 2: Bitstream (optional, FPGA config)
├─ Partition 3: U-Boot (second-stage bootloader)
├─ Partition 4: Device Tree Blob (optional)
└─ Authentication Certificates (if secure boot)
```

### BIF File (Boot Image Format)

```bash
# boot.bif for Zynq-7000 (simple)
the_image:
{
    [bootloader] /path/to/fsbl.elf
    /path/to/system.bit          // FPGA bitstream (optional)
    /path/to/u-boot.elf          // U-Boot proper
}

# boot.bif for Zynq MPSoC (with PMUFW, ATF)
the_image:
{
    [bootloader, destination_cpu = a53-0] /path/to/fsbl.elf
    [pmufw_image] /path/to/pmufw.elf
    [destination_cpu = a53-0, exception_level = el-3, trustzone] /path/to/bl31.elf
    [destination_cpu = a53-0, exception_level = el-2] /path/to/u-boot.elf
    /path/to/system.bit          // FPGA bitstream
}
```

```bash
# Generate BOOT.BIN
bootgen -image boot.bif -arch zynq -o BOOT.BIN -w on

# For MPSoC
bootgen -image boot.bif -arch zynqmp -o BOOT.BIN -w on
```

### Zynq-7000 BOOT.BIN on SD Card

The Zynq-7000 Boot ROM expects `BOOT.BIN` at the root of the first FAT partition:

```
/dev/mmcblk0
├─ /dev/mmcblk0p1  [FAT32, 256 MB, label: BOOT]
│   ├─ BOOT.BIN          (boot image: FSBL + bitstream + U-Boot)
│   ├─ uEnv.txt          (U-Boot environment)
│   ├─ boot.scr          (boot script)
│   ├─ zImage            (Linux kernel)
│   ├─ devicetree.dtb    (device tree)
│   └─ uramdisk.image.gz (initramfs, optional)
└─ /dev/mmcblk0p2  [ext4, remainder, label: rootfs]
    └─ Full Linux root filesystem
```

### Zynq MPSoC BOOT.BIN on SD Card

```
/dev/mmcblk0
├─ /dev/mmcblk0p1  [FAT32, 256 MB]
│   ├─ BOOT.BIN          (FSBL + PMUFW + ATF + U-Boot + bitstream)
│   ├─ boot.scr
│   ├─ Image             (ARM64 kernel)
│   ├─ zynqmp-zcu102.dtb
│   └─ initrd.img
└─ /dev/mmcblk0p2  [ext4, remainder]
    └─ rootfs
```

> **Versal note**: Versal uses a different boot image format (PDI — Programmable Device Image). The `v++` or `bootgen` tool generates `.pdi` files. The PMC ROM loads the PDI, which contains the PLM and all subsequent partitions.

## Secure Boot on Xilinx SoCs

Xilinx secure boot provides authentication and encryption for the boot image, protecting both the software chain and the FPGA bitstream.

### Security Stack Comparison

| Feature | Zynq-7000 | Zynq MPSoC | Versal |
|---|---|---|---|
| **Authentication** | RSA-2048 | RSA-4096 | RSA-4096 or ECDSA P-384 |
| **Encryption** | AES-256 (CBC) | AES-256-GCM | AES-256-GCM |
| **Key storage** | eFuse (BBRAM for AES) | eFuse + PUF | eFuse + PUF |
| **Anti-rollback** | No | Yes (eFuse counter) | Yes |
| **PUF** | No | Yes (device-unique key) | Yes |
| **JTAG disable** | Yes (eFuse) | Yes (full debug lock) | Yes |
| **HSM support** | No | Yes (external HSM for key injection) | Yes |

### Chain of Trust

```
Boot ROM (hashes in eFuse)
    │ verifies RSA signature
    ▼
FSBL (encrypted + authenticated)
    │ decrypts + verifies
    ▼
U-Boot / ATF (encrypted + authenticated)
    │ decrypts + verifies
    ▼
Bitstream (encrypted + authenticated)
```

### Bootgen with Authentication

```bash
# Generate RSA key pair
openssl genrsa -out rsa_priv.pem 2048   # Zynq-7000
openssl genrsa -out rsa_priv.pem 4096   # MPSoC

# BIF with authentication
# authenticated.bif
the_image:
{
    [pskfile] rsa_priv.pem
    [sskfile] rsa_priv.pem
    [bootloader, authentication = rsa] /path/to/fsbl.elf
    [authentication = rsa] /path/to/u-boot.elf
    [authentication = rsa] /path/to/system.bit
}

# Generate authenticated BOOT.BIN
bootgen -image authenticated.bif -arch zynqmp -o BOOT.BIN
```

### Bootgen with Encryption

```bash
# Generate AES key
openssl rand -hex 32 > aes_key.nky

# BIF with encryption
# encrypted.bif
the_image:
{
    [aeskeyfile] aes_key.nky
    [bootloader, encryption = aes] /path/to/fsbl.elf
    [encryption = aes] /path/to/u-boot.elf
    [encryption = aes] /path/to/system.bit
}

# Generate encrypted BOOT.BIN
bootgen -image encrypted.bif -arch zynqmp -o BOOT.BIN
```

### eFuse Programming

```bash
# Zynq MPSoC eFuse programming (IRREVERSIBLE)
# Program RSA public key hash
xilinx-bootgen -p efuse -sha_rsa sha3_384 -efuseppkhash ppk_hash.txt

# Enable secure boot (once set, only authenticated images boot)
# This is a ONE-WAY operation
```

> **Warning**: Programming the `SEC_LOCK` eFuse bit permanently disables JTAG and forces secure boot. There is no recovery. Always test the full secure boot flow on a sacrificial board before programming production units.

### ZedBoard (Zynq-7000)

| Setting | Value |
|---|---|
| Boot mode | SD card (MIO straps) |
| UART | UART1 at 115200 baud (USB-UART) |
| BOOT.BIN location | FAT partition root |
| FPGA config | PCAP, `.bit.bin` file |

| Key docs | ZedBoard Getting Started, UG585 |

### ZCU102 (Zynq UltraScale+ MPSoC)

| Setting | Value |
|---|---|
| Boot mode | SD card or QSPI (switch S1) |
| UART | UART0 at 115200 baud |
| BOOT.BIN | Contains FSBL + PMUFW + ATF + U-Boot |
| FPGA config | PCAP via CSU DMA |

| Setting | Value |
|---|---|

| Setting | Value |
|---|---|

| AI Engines | Configured by PLM after PS boot |

## Common Xilinx-Specific Boot Problems

### "BOOT.BIN not found" / Boot ROM Hang

| Check | Action |
|---|---|
| Boot mode straps | Verify MIO pin voltages with DMM. MIO[5:3] sets boot mode |
| BOOT.BIN on FAT | Ensure `BOOT.BIN` is in root of first FAT partition. Filename is case-sensitive |
| SD card format | Use FAT32 (not exFAT). Some Boot ROMs fail with exFAT |
| FSBL in BOOT.BIN | Verify with `bootgen -verify BOOT.BIN` |

### "FSBL runs but hangs at DDR init"

| Cause | Fix |
|---|---|
| Wrong DDR parameters | Regenerate `ps7_init.c` / `psu_init.c` from Vivado with correct DDR part number |
| DDR not populated | Some dev boards have optional DDR. Check BOM |
| Temperature | DDR calibration may fail at temperature extremes. Widen margins |

### "FPGA config fails in U-Boot"

| Cause | Fix |
|---|---|
| Wrong bitstream format | U-Boot needs `.bin` (raw), not `.bit` (with header). Use `-bin_file` in Vivado |
| Bitstream too large | Check bitstream size against available OCM/DDR. Large bitstreams need DMA |
| PCAP not enabled | Ensure `devcfg` driver is compiled in U-Boot (`CONFIG_FPGA_XILINX`) |
| FPGA already configured | Check with `fpga info 0`. If configured, reset PL first |

### "Kernel panic: VFS: Unable to mount root fs"

| Cause | Fix |
|---|---|
| Wrong root device | `root=/dev/mmcblk0p2` may need to be `root=/dev/mmcblk1p2` if SD is not mmc 0 |
| Missing `rootwait` | SD detection takes time. Always add `rootwait` |
| ext4 not supported | Ensure ext4 filesystem driver is compiled into kernel or as module |

### "ATF boot failure on MPSoC"

| Cause | Fix |
|---|---|
| BL31 not in BOOT.BIN | MPSoC requires ARM Trusted Firmware (BL31) at EL3. Include in BIF |
| Wrong exception level | U-Boot must be compiled for EL2 (hyp mode), BL31 for EL3 |
| PMUFW missing | CSU ROM requires PMUFW before loading FSBL. Include in BIF |
