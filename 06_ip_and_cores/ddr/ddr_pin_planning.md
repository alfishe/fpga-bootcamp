# DDR PCB Routing & Pin Planning

DDR memory is the highest-speed parallel bus on most FPGA boards. Getting the pin assignment and PCB layout right is a prerequisite for the controller calibration to succeed. This article covers the physical-layer rules that apply across all vendors.

---

## DDR Bus Signal Groups

```
┌────────────────── DDR Interface ──────────────────┐
│                                                    │
│  Clock Group:   CK_P, CK_N    (differential)       │
│  Address/Cmd:   A[15:0], BA[2:0], RAS_N, CAS_N,    │
│                 WE_N, CS_N, CKE, ODT               │
│  Data Group 0:  DQ[7:0], DQS_P[0], DQS_N[0], DM[0] │
│  Data Group 1:  DQ[15:8], DQS_P[1], DQS_N[1], DM[1] │
│  ...                                               │
│  Control:        RESET_N, ZQ (240Ω reference)       │
└────────────────────────────────────────────────────┘
```

Each data group (byte lane) must be routed as a matched-length group. The address/command/control group must be length-matched to the clock. The clock itself must arrive at each DRAM within one clock period of the data.

## Topology: Fly-By vs T-Branch

### Fly-By (DDR3/DDR4 — Required)

```
FPGA ──→ CK ──┬──────┬──────┬──────┬──────→ VTT
              DRAM 0  DRAM 1  DRAM 2  DRAM 3

FPGA ──→ A/Ctrl (daisy-chain to all DRAMs)

FPGA ◄── DQ/DQS ──┬──────┬──────┬──────┬──────
                   DRAM 0  DRAM 1  DRAM 2  DRAM 3
                    (each has dedicated point-to-point DQ/DQS)
```

- **CK and Address/Command**: Daisy-chained. Arrives at each DRAM with increasing delay → requires **write leveling** to compensate
- **DQ/DQS**: Point-to-point per byte lane (NOT daisy-chained)
- Fly-by is MANDATORY for DDR3 and DDR4 at speeds above 800 MT/s
- Write leveling in the controller compensates for CK-to-DQS skew at each DRAM

### T-Branch (DDR2, Some DDR3 Below 800 MT/s)

```
FPGA ──→ CK ──┬──────┬──────┬──────┬──────
              DRAM 0  DRAM 1  DRAM 2  DRAM 3
         (equal-length branches from a central point)
```

- All signals branch from a common point with equal segment lengths
- No write leveling required (all DRAMs see CK aligned)
- **Not recommended for DDR3 ≥ 800 MT/s** — stub reflections degrade signal integrity

## Pin Assignment Rules

### FPGA-Side Rules (Universal)

| Rule | Why |
|---|---|
| **DQS_P must be on a clock-capable pin** (MRCC/SRCC on Xilinx, dedicated DQS on Intel) | DQS is used as a clock for the input SERDES |
| **All DQ bits in a byte lane must be in the same I/O bank** | They share a common DQS, clock, and VREF |
| **DM must be in the same bank as its byte lane** | DM is captured on the same DQS edge |
| **Address/command can span banks** | Not clocked by DQS; uses CK directly |
| **CK_P must be on a dedicated clock output** | Must drive a low-skew global clock net |

### Bank Voltage Compatibility

| DDR Type | I/O Standard | VCCIO | VREF |
|---|---|---|---|
| **DDR2** | SSTL18 | 1.8V | 0.9V |
| **DDR3 / DDR3L** | SSTL15 | 1.5V (1.35V for DDR3L) | 0.75V (0.675V for DDR3L) |
| **DDR4** | POD12 | 1.2V | Internal (VREFDQ calibration) |
| **LPDDR4** | LVSTL (0.6V or 0.5V) | 1.1V/1.8V (VDD1/VDD2) | Internal only |

The I/O bank containing DDR signals must be powered at the DDR I/O voltage. This means that bank CANNOT be shared with LVCMOS 3.3V or other standards.

## PCB Routing Requirements

### Length Matching Tolerances

| Signal Group | Tolerance (DDR3-1600) | Tolerance (DDR4-3200) |
|---|---|---|
| **DQ to DQS** (within a byte lane) | ±10 ps (±1.5 mm on FR4) | ±5 ps (±0.75 mm on FR4) |
| **Byte lane to byte lane** (DQ groups) | ±50 ps (±7.5 mm) | ±20 ps (±3 mm) |
| **CK to Address/Command** | ±25 ps (±3.75 mm) | ±10 ps (±1.5 mm) |
| **CK to DQS** (skew budgeted by controller) | Controller dependent (write leveling) | Controller dependent |

### Impedance

| Signal | Characteristic Impedance | Termination |
|---|---|---|
| **DQ, DQS, DM** | 50Ω (single-ended), 100Ω (differential for DQS) | ODT at DRAM (Rtt_Nom) |
| **CK** | 100Ω differential | 100Ω differential at far end (for fly-by) |
| **Address/Command** | 50Ω single-ended | VTT termination at far end (e.g., 39Ω to VTT = VDD/2) |

### VREF

- **DDR3**: External VREF = VDD/2, typically generated by a voltage divider or dedicated regulator. Must be decoupled with a 0.1 µF + 0.01 µF capacitor at each VREF pin.
- **DDR4**: Internal VREF — the DRAM generates its own reference via VREFDQ calibration. No external VREF supply needed.

## Board-Level Checklist

```
□ DDR I/O bank powered at correct VCCIO
□ VTT termination supply (VDD/2) — typically a DDR termination regulator (e.g., TPS51200)
□ VREF decoupling at each FPGA VREF pin
□ ZQ calibration resistor: 240Ω ±1% from ZQ pin to GND
□ CK differential pair: length-matched within ±1 ps (tight)
□ CK-to-address/command matching within spec
□ DQS_P on clock-capable pin; verify in FPGA pin table
□ All DQ/DQS/DM of one byte lane in same I/O bank
□ Fly-by topology for DDR3 ≥ 800 MT/s (no T-branch)
□ No stubs on high-speed signals (DDR4 especially)
□ DDR RESET_N driven by FPGA GPIO (not tied HIGH)
□ Adequate ground plane under all DDR routing (solid reference)
□ Keep DDR routing away from noisy sources (SMPS, clock oscillators)
```

## Vendor-Specific Pin Planning Notes

| Vendor | Tool | Key Command |
|---|---|---|
| **Intel** | Pin Planner (Quartus) | Assign DQS pins first; use "Show DQ/DQS Pins" filter |
| **Xilinx** | MIG Pin/Bank Selection | Validates I/O standard per bank; rejects pin swap violations |
| **Lattice** | Diamond Pin Planner | DQS pins pre-assigned; use IPexpress to generate constraints |
| **Microchip** | Libero I/O Editor | Pin assignment wizard for FDDR/MDDR controllers |
| **Gowin** | Gowin EDA FloorPlanner | Manual assignment; refer to device pinout for DQS groups |
